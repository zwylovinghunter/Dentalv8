# 牙齿病变目标区域识别与辅助分析平台

本项目是一个基于 Gradio Blocks 的课程验收与科研展示平台，用于牙齿或口腔图像中疑似病变目标区域的辅助识别、模型对比、智能问答和报告生成。

## 启动方式

```bash
python app.py
```

启动后在浏览器中打开 Gradio 页面即可使用。

## 主要功能

- 首页 Dashboard：展示累计检测任务、真实检测目标数量、失败次数、模型平均耗时、平均置信度和最近检测摘要。
- 首页 Dashboard：支持一键清空所有历史记录并恢复初始统计状态。
- 图像检测：上传图像后选择模型，设置置信度阈值和 IoU 阈值，运行 YOLO CPU 推理。
- 多模型对比：同一张图像一键运行三个模型，并展示对比表和自动总结。
- 智能问答助手：基于当前检测结果 JSON 或多模型对比 JSON 回答问题。
- 报告中心：生成 Markdown 报告，可在页面预览并下载。
- 项目说明：说明系统用途、权重发现、CPU 推理和使用声明。

## 已有权重使用方式

系统会自动递归扫描当前项目中的 `.pt` 文件，优先使用 `results/**/weights/best.pt`，并结合目录名、README、args.yaml 和文件名关键词自动匹配三个模型：

- 均衡型基线模型：匹配 `results/yolov8n+baseline_e50/weights/best.pt`。
- 高精度牙齿病变定位模型：匹配 `yolov8m+PIoU`。
- 高召回轻量化牙齿病变检测模型：匹配 `yolov8n+SPDConv-neck-P4_cosLR`。

如果某个模型未匹配到权重、权重加载失败或推理失败，系统会显示明确失败原因，不生成伪造检测框，也不会让单个模型失败影响其他模型。

## CPU 推理说明

本平台面向当前 CPU 环境设计，所有 YOLO 推理均显式使用 CPU。Dashboard 会显示当前运行设备状态。

## 智能问答说明

演示版本使用代码内配置的 API Key，正式部署建议改为环境变量或密钥管理服务。README 不展示真实 Key。

如果云端大模型接口不可用、网络异常、鉴权失败、模型不可用或响应字段缺失，系统会自动降级为本地规则问答，不影响检测、对比和报告功能。

## 报告与历史记录

报告默认保存到项目相对目录 `outputs/reports/`，文件名自动生成，避免覆盖已有报告。

历史记录默认保存到 `outputs/history.json`。如果历史文件不存在会自动创建；如果历史文件损坏会自动恢复为空记录。

Dashboard 统计口径：

- 累计检测图片数按用户上传图像任务计数，不按模型数量重复计数。
- 累计检测目标数只统计成功真实推理生成的检测框。
- 平均置信度、平均耗时和最快模型只统计成功真实推理结果。
- 失败任务单独统计失败次数。

## 依赖安装

如当前环境缺少依赖，可执行：

```bash
pip install -r requirements.txt
```

## 使用声明

本系统仅用于牙齿病变疑似区域的辅助识别与科研展示，不作为临床诊断依据，最终结果应由专业人员复核。
